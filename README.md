# SECTA Feedback — система обратной связи для барбершопа

NPS-опросник с интеллектуальной классификацией клиентов через CRM API. Реальный продакшен-проект для барбершопа SECTA (Нижний Новгород): клиенты сканируют QR в зале, проходят персонализированный опрос, ответы записываются в облачную БД, владелец видит аналитику в дашборде.

> **Не учебный проект.** Реальный бизнес, реальные клиенты, реальные деньги на инфраструктуру. Запущен в продакшен в мае 2026.

---

## Что внутри

- **6 типов опросных форм** под разные сегменты клиентов (постоянные, новички, сопровождающие и т.д.) — каждый видит вопросы только под свой профиль
- **Классификация на лету** через yclients API: телефон → история визитов → подходящий тип формы
- **Бэкенд на Yandex Cloud Functions** — serverless, без серверов и обслуживания
- **Хранение в YDB Serverless** — 30+ плоских колонок для аналитики
- **Аналитика в DataLens** — владелец видит NPS, оценки мастеров, тексты отзывов в реальном времени
- **174 теста, 94% покрытия** — pytest, моки, edge-кейсы
- **Миграция с FastAPI/Render на Yandex Cloud** — целиком, с документированием решений

## История миграции v1 → v2

Первая версия жила на **FastAPI + Render** + **Tally-формы**. На бумаге работала, но в реальном продакшене в РФ выяснились блокирующие проблемы:

- **152-ФЗ.** Тrans-border transfer персональных данных (телефоны клиентов уходили на серверы за рубежом) — нарушение, штрафы для оператора ПД до 18 млн ₽.
- **Доступность.** Tally без VPN не открывался в РФ — клиенты в зале просто не могли пройти опрос.
- **Render** периодически ограничивал доступ из РФ — продакшен мог упасть без предупреждения.
- **Нет своей БД** → нет аналитики → данные оставались внутри Tally и были бесполезны для владельца.

**Решение:** мигрировать на полностью российский стек.

| | v1 (legacy) | v2 (current) |
|---|---|---|
| Compliance | Транс-border ПД (нарушение 152-ФЗ) | Все данные в РФ, оператор ПД зарегистрирован в РКН |
| Доступность из РФ | Через VPN | Нативно |
| Бэкенд | FastAPI на Render | Yandex Cloud Functions |
| БД | — | YDB Serverless |
| Формы | Tally | Кастомные HTML на Tilda |
| Аналитика | — | DataLens |
| Покрытие тестами | 113 тестов | 174 теста, 94% |

Перед миграцией: проверил карточку оператора ПД в реестре РКН, спроектировал схему БД с TTL=365 дней (требование 152-ФЗ), убедился что все компоненты (Tilda, Yandex Cloud, YDB) — российские.

История миграции в `git log` — два больших коммита, видна эволюция.

## Чему научил проект

- **Compliance — это часть архитектуры, а не отдельная задача.** 152-ФЗ заставил переехать с привычного западного стека (Render, Tally) на российский (Yandex Cloud, Tilda) — и это сделало проект надёжнее, а не дороже.
- **Продуктовое мышление, а не код ради кода.** Каждое решение продиктовано задачей: транс-border ПД ломал закон → миграция, JSON в БД ломал аналитику → плоские колонки, 5+ типов клиентов → классификатор на бэке.
- **Тесты как контракт при миграции.** При переезде они сначала сломались, потом стали guardrail-ом — без них переезд был бы рулеткой.
- **Serverless дешевле и удобнее VPS для маленьких проектов** — за месяц инфраструктура стоит в районе нуля рублей.
- **Компромисс между правильным (UTC в БД) и удобным (МСК в БД) — это нормально**, главное явно зафиксировать выбор и причину.
